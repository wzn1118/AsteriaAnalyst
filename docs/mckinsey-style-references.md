## McKinsey-related open-source references

- [mckinsey/vizro](https://github.com/mckinsey/vizro)
  Why it matters:
  McKinsey open-source dashboard/app framework. Useful as a reference for executive-facing analytical presentation and structured dashboard delivery.

- [mckinsey/causalnex](https://github.com/mckinsey/causalnex)
  Why it matters:
  McKinsey open-source causal reasoning toolkit. Useful when future versions need more consulting-style causal hypothesis framing instead of pure descriptive analytics.

- [mckinsey/qbstyles](https://github.com/mckinsey/qbstyles)
  Why it matters:
  McKinsey open-source plotting style package. Useful as a visual reference for cleaner, more presentation-ready statistical figures.

- [mckinsey/agents-at-scale-ark](https://github.com/mckinsey/agents-at-scale-ark)
  Why it matters:
  McKinsey open-source agent orchestration project. Useful as an agent-systems reference for multi-step report generation and reusable prompt/skill pipelines.

## How Asteria uses these references

- Historical report rewriting:
  Prefer answer-first, SCQA, and pyramid-style flow when the provided template resembles a consulting report.

- Market analysis:
  Prefer executive summary -> market structure -> drivers -> risks -> next actions instead of generic EDA ordering.

- Dashboard handoff:
  Produce BI-friendly artifacts and dashboard specs rather than only raw notebooks or tables.
