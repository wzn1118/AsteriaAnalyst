# Asteria Analyst 全部可用功能目录

> 这是一份面向公开发布版的功能清单，列出用户可通过页面、随附启动器、显式本地配置或已注册本地 API 完成的能力；范围覆盖当前发布版的可操作功能。
>
> 每个条目都说明入口、需要准备的内容、可执行的动作、完成后得到的结果，以及对应的可用条件。Skill、Report Agent Team、Codex Runtime 等扩展均按实际启用条件列出。

## 1. 如何阅读这份目录

### 1.1 可用性状态

| 状态 | 含义 | 用户应如何理解 |
| --- | --- | --- |
| 默认可用 | 本机服务启动后即可进入或查询的基础能力 | 有效数据和合理输入决定每项任务的执行结果。 |
| 数据条件可用 | 需要上传或选择适配的数据、字段、工作表或方法 | 系统会显示准备度或阻断原因，用户仍需确认业务口径。 |
| 可选配置 | 依赖本机 AI 提供方、`Rscript` 或特定服务端配置 | 配置完成后可用；页面会呈现配置状态或降级结果。 |
| 默认关闭的本地扩展 | 需要可信本机管理员显式开启环境变量 | 面向本机受控工作区，通过受限接口执行已批准的任务。 |
| 正式受控 | 正式报告链路和质量门禁确认后交付 | 正式管理报告由完整 AI 链路、确定性计算、证据验证和发布门禁确认。 |

### 1.2 本目录覆盖的范围

1. 六个浏览器工作区：`/`、`/analysis`、`/lab`、`/lab/method-guide`、`/revision`、`/revision/workspace`。
2. 数据、历史报告、业务背景、受控统计、自动分析、智能报告、报告资产和正式 PDF 门禁。
3. Analysis Lab 的字段建模、派生指标、方法卡、对象范围、多运行实例、方法包、四种执行模式和 PDCA。
4. 报告检索、修订会话、事件、预览、附件、批注、差异和本地修订版本发布。
5. AI、R、Skill、Feature Trial、Report Agent Team、Codex Runtime、进程、管线、学习账本和生态目录。
6. 已注册本地诊断或管理员 API。目录以 API 入口标注，页面入口和接口入口分别列出。

`CodeExecutionRequest`、`CodeEditorPanel` 和其他内部名称属于实现层。公开用户功能以本目录列出的注册入口与页面操作为准；脚本执行策略由本地服务端受控配置。

| --- | --- | --- | --- | --- | --- |
| 查看已保存数据集 | `/`、`/analysis`、`/lab` 或 `GET /api/datasets` | 本机已有上传数据 | 搜索、选择和复用数据集 | 数据集标识、文件信息、工作表摘要和画像入口 | 空目录是正常初始状态。 |

| 查看数据画像 | 选择数据集后或 `GET /api/datasets/{dataset_id}` | 已存在的数据集 | 查看行数、列数、类型、缺失、唯一值、样本和候选字段角色 | 可用于后续方法选择的结构化摘要 | 读取完成后确认表头、时间、单位、主键和分析粒度。大数据集预览可能是采样结果。 |

| 获取数据工作流建议 | `GET /api/datasets/{dataset_id}/workflow` | 已存在的数据集与当前工作表 | 查看多表指纹、候选关系、质量告警和路线建议 | 工作流蓝图与建议 | 用户确认连接键、时间粒度、汇总口径和业务定义；系统提供关系与质量建议。 |

## 4. 数据理解、受控统计与自动分析

### 4.2 受控统计和图表

| 功能 | 入口 | 需要的输入 | 可执行动作 | 成功后的结果 | 状态与边界 |
| --- | --- | --- | --- | --- | --- |
| 统计方法目录 | `/lab` 或 `GET /api/statistics/catalog` | 可选数据上下文 | 搜索、分类、筛选和查看方法说明 | 方法类别、适用条件、字段角色、状态和阻断原因 | 目录以 `live`、`catalog`、`planned`、`learned` 等状态标记方法，数据与字段条件决定执行范围。 |
| 描述、分布和 KPI | `/analysis` 或 `POST /api/statistics/run` | 数值/分类字段、必要时排序或阈值 | 运行描述统计、频数、分位数、截尾均值、Winsorize、Gini、Pareto、分组 KPI 等 | 统计表、分布摘要、排序或结构结果 | 输出样本分布和 KPI；因果解释结合研究设计与业务证据。 |
| 分组、交叉与比较 | 同上 | 指标、维度、分组字段 | 运行交叉表、透视、分组汇总、A/B、t 检验、ANOVA、卡方、Fisher、非参数或重采样分析 | 差异、区间、列联表或显著性结果 | 分组必须由有效字段定义；应检查样本稀疏、独立性和实验设计。 |
| 关系、回归和预测 | 同上 | 目标、特征、必要时分组或筛选条件 | 运行相关、OLS、岭/Lasso/Elastic Net、Logit、Poisson、随机森林等 | 系数、拟合、诊断、预测或特征信息 | 因果结论需要额外的研究设计；字段类型、样本量和方法前提决定可运行的方法。 |
| 探索、聚类与时序 | 同上 | 适配字段、参数、时间字段或窗口 | 运行 PCA、K-Means、基础神经网络、稳定性和时序类方法 | 成分、簇、模型摘要、图表或诊断 | 数据、依赖、参数和方法前提共同决定可运行性。 |
| 图表、代码说明和计算证据 | 统计或自动分析结果区 | 已完成的分析运行 | 查看后端返回的图表规格、表格、模型解释和证据工件 | 可审阅图表、计算说明、证据表与下载链接 | 浏览器集中展示后端产出的分析说明、图表和证据工件。 |

`POST /api/statistics/run` 的常用字段包括 `dataset_id`、`analysis_type`、`active_sheet`、`target`、`features`、`group_column`、`group_a`、`group_b` 以及方法参数。具体字段和每一种方法的要求见 [API 参考](api-reference.zh-CN.md) 与 [逐项功能参考](function-reference.zh-CN.md#6-受控统计图表代码与证据)。

### 5.2 同步、异步报告任务和资产

### 5.3 正式管理报告发布门禁

正式 `management_report.pdf` 通过以下强制链路发布：

`raw data` -> `DataProfileService` -> `AIFieldSemanticMapper` -> `AIBusinessContextRouter` -> `AIMetricDerivationPlanner` -> `DeterministicMetricExecutor` -> `EvidenceValidator` -> `ReportBindingLayer` -> `FormalPDFReleaseGate` -> `management_report.pdf`

探索性或非正式工作流按需配置 AI 提供方；正式 `management_report.pdf` 按链路保留字段语义映射、业务上下文路由和指标推导规划的 AI trace 与 schema 校验。

| 功能 | 用户看到的结果 | 必须满足的条件 | 条件尚待补齐时的输出 |
| --- | --- | --- | --- |
| 查看正式报告资格 | 任务的 trace、schema、确定性计算、证据、质量与门禁状态 | AI 三个规划步骤均有 trace 且 schema 有效；数值由确定性执行器产生；证据校验通过 | 任务输出普通资产或 `debug_report`，供链路诊断和条件补齐。 |
| 发布正式 `management_report.pdf` | 通过门禁的正式 PDF 与对应工件 | `FormalPDFReleaseGate` 批准且最终质量分为 `90` 或更高 | 全部条件满足后发布正式 PDF；trace、schema、证据、数值链路与质量状态在资格页显示。 |

详见 [AI 强制发布链](architecture_ai_mandatory_chain.zh-CN.md) 和 [报告可信机制](report-integrity.zh-CN.md)。

## 10. 建议的使用路径

1. **基础本地分析**：启动服务 -> 上传 `.csv`/`.tsv`/`.xlsx`/`.dta` -> 检查画像和工作表 -> 在 `/analysis` 运行适配统计 -> 审阅表、图和证据 -> 下载实际生成的资产。
2. **方法探索与复核**：进入 `/lab` -> 绑定字段角色和对象范围 -> 选择方法或方法包 -> 单独/批量运行 -> 比较 `run_id`、准备度、阻断原因和结果工件 -> 必要时保存方法卡。
3. **报告交付**：补充业务简报和材料 -> 设计多表、读者、深度、风格和交付偏好 -> 使用同步或异步报告作业 -> 查看资产与质量状态 -> 只有门禁全部通过时才将 PDF 视为正式管理报告。
4. **已有报告修订**：在 `/revision` 找到报告 -> 创建会话 -> 添加修订说明、附件和批注 -> 检查时间线、预览和 diff -> 满足服务端条件时发布本地修订版本。
5. **本地扩展**：先验证 Runtime 或 `Rscript` 状态 -> 由可信管理员显式开启开关 -> 导入并审阅 Skill/Team -> 用 Feature Trial 验证数据适配性 -> 再运行受控的 Team 或 Runtime 任务。

## 11. 公开发布边界

- 公开仓库包含源代码、测试、启动器、示例、文档和经脱敏的演示材料。
- 客户数据、真实业务报告、API Key、访问令牌、运行日志、私有工作区、未授权第三方材料和可反推客户身份的信息保留在受保护的本地位置。
- 本地 HTTP 服务默认面向回环地址；共享网络与互联网使用独立部署架构、身份认证和访问控制。
- 本地修订版发布登记本机报告版本；GitHub Release 由仓库维护流程处理。

更多安全与部署细节见 [安全与部署边界](security-deployment.zh-CN.md)。
