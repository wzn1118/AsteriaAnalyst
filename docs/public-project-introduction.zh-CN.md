# Asteria Analyst 公开项目介绍

> 从一份表格开始，到一组可复核的交付资产结束。

Asteria Analyst 是面向数据团队、运营团队、研究团队与咨询交付团队的本地优先分析交付工作台。它把 CSV、Excel 等结构化业务数据组织为一条可审阅的工作流：理解数据和问题、规划指标与方法、执行确定性计算、沉淀证据和交付物、处理修改意见，并保留可以继续使用的版本上下文。

它的目标是将分析过程沉淀为团队可检查、讨论、交接和复用的材料。

需要逐项查看页面、功能、后端职责模块、启用条件和安全边界时，先阅读 [全部可用功能目录](feature-catalog.zh-CN.md)。它按入口、输入、动作、结果和边界列出公开版全部可用功能；再结合 [模块与功能手册](module-guide.zh-CN.md) 理解完整工作流，或使用 [功能逐项参考](function-reference.zh-CN.md) 核对接口字段。

## 它解决的工作断点

在经营复盘、专题分析和咨询交付中，团队经常遇到同一类断点：

| 工作断点 | 常见后果 | Asteria 的处理方式 |
| --- | --- | --- |
| 数据字段含义不清 | 同一字段被不同人按不同口径理解 | 先生成数据画像，再记录字段语义与分析对象 |
| 业务问题没有转成可执行分析 | 图表很多，但回答不了决策问题 | 将业务背景、受众、指标和方法组织为分析计划 |
| 数值、结论和报告脱节 | 结论无法复核，修改时需要反复返工 | 数值由确定性执行层计算，报告绑定已验证的证据 |
| 交付物只有一份静态文档 | 管理层、分析师和审阅者只能各自重做材料 | 同时产出阅读材料、图表表格、结构化结果和交付清单 |
| 修改意见散落在聊天记录中 | 无法知道哪个版本改了什么、为什么改 | 在修订工作区管理批注、附件、差异和版本发布 |

## 产品承诺

### 输入变资产

用户从数据集、工作表和业务问题开始。系统先帮助确认数据粒度、字段类型、空值、时间范围、单位和候选指标，再进入分析。

### 分析变证据

字段语义、业务路由和指标规划可以使用 AI 辅助，但正式数值由确定性计算层执行。图表、表格、方法结果和结构化输出与相应证据一同保留，方便复核。

### 证据变材料

同一批分析结果可以服务不同阅读者：管理层可阅读决策摘要，分析师可检查方法结果和表格，工程或审计伙伴可查看结构化资产、trace 和 manifest。

### 材料变版本

报告支持持续修订：可进入修订中心，通过批注、附件、过程记录和差异查看完成后续修改，并在审核后发布新版本。

## 从数据到交付的工作流

```mermaid
flowchart LR
    A["数据与业务背景"] --> B["数据画像与字段语义"]
    B --> C["业务路由与指标规划"]
    C --> D["确定性统计与计算"]
    D --> E["证据、图表与报告部件"]
    E --> F["质量检查与下载资产"]
    F --> G["批注、修订与版本发布"]
```

### 1. 数据与业务背景

导入或选择数据集后，明确四件事：

1. **分析对象**：一行数据代表订单、客户、商品、门店、事件还是某个时间周期。
2. **业务问题**：希望解释什么、比较什么、优先做什么决策。
3. **读者和用途**：管理会议、经营复盘、研究分析、客户交付或内部方法验证。
4. **约束条件**：时间粒度、计算口径、敏感字段、历史模板和可用数据范围。

### 2. 数据画像与字段语义

在执行分析前，先检查结构性事实：工作表是否正确、字段类型是否混合、缺失是否影响指标、币种或单位是否一致、时间范围是否足够、主键或分组字段是否可信。

AI 可以辅助解释字段名和候选业务角色。用户在关键口径上完成业务确认，尤其是指标定义、单位、时间窗和数据粒度。

### 3. 业务路由与指标规划

系统将业务背景与数据结构一起用于选择适合的问题框架、指标和方法。规划阶段需要区分：

- 已被数据直接支持的指标；
- 需要从字段推导的指标；
- 只能作为假设或待验证问题的判断；
- 需要补充数据或完成验证后进入结论的内容。

这一步把“想看什么”转为可解释的分析安排，将指标、方法、假设和验证要求分别记录。

### 4. 确定性计算与方法执行

关键数值由确定性执行器或选定的方法计算。Analysis Lab 可用于方法探索、比较和方法级输出；正式管理报告使用受控链路，要求完整的 AI trace、schema 校验、证据校验和质量门禁。

### 5. 交付、审阅与续改

成功的工作流可以生成面向阅读、复核和再利用的资产，例如报告内容、图表、表格、CSV/XLSX、HTML、JSON、README、manifest 或方法级证据文件。实际文件取决于所选工作流和本次运行的交付清单。

### 正式分析具体包含什么

1. **数据接入与工作簿选择**：支持 CSV、TSV、Excel <code>.xlsx</code>、Stata <code>.dta</code>。旧版 Excel <code>.xls</code> 先转换为 <code>.xlsx</code>；部分浏览器控件会显示该扩展名，分析数据使用后端支持的格式。成功导入后可检查行列规模、字段类型、缺失、样本、工作表和活动工作表。多表任务由用户确定单表、合并、分别或组合策略，并记录连接键和汇总口径。
2. **业务简报与材料上下文**：可以关联历史报告和业务背景，并填入分析对象、时间范围、核心问题、目标读者、交付用途、期望结果和限制。材料进入路线选择与报告设计，当前结论绑定已验证的事实和数值证据。
3. **受控统计与图表**：可配置描述统计、分组/交叉、相关、回归、组间检验、分类检验、非参数与重采样、PCA、K-Means、随机森林、基础时序等适配方法。结果可能包含数值表、模型摘要、诊断、图表规格和计算证据；方法状态、数据和字段条件决定本次可运行范围。
4. **报告任务与资产**：提交后可查看阶段、进度、结构化章节、关键指标、图表、表格和后端实际返回的文件。实际资产包括 HTML、Markdown、CSV、XLSX、JSON、manifest、方法证据或报告文件，交付清单随本次数据和路线确定。

### Analysis Lab 具体包含什么

1. **字段和派生指标**：先读数据画像，再把字段绑定为目标、分组、标签、时间、特征、对象或图形字段；需要时明确派生指标的名称、依赖字段和计算口径。
2. **方法卡和运行实例**：每种方法会展示适配条件、可运行性和阻断原因。用户可按字段、全量行或重点对象创建独立实例，防止同一方法在不同对象和参数下的结果相互覆盖。
3. **执行与复核**：可单独运行、批量运行或把多个结果合并解释。页面显示每个实例的状态和实际返回的摘要、指标、图表、表格、文本、审计信息、下载资产或可重跑的报告部件蓝图。
4. **正式交付**：Lab 用于探索、验证和方法比较。正式管理报告由完整 AI 强制链、确定性计算、证据校验和发布门禁交付。

### 修订工作区具体包含什么

1. **修订输入**：文本修改意见、快捷任务、批注和附件都可以成为会话输入；批注工具包括浏览、画笔、矩形、高亮和橡皮。
2. **过程与结果**：可以查看事件流、轮询状态、HTML/PDF 预览、文件差异、任务状态、附件、产物和已发布本地版本。启用可选运行时后，相关任务还可能返回 CLI 运行、日志、摘要和恢复信息。
3. **本地发布的含义**：修订工作区将通过当前会话条件的版本登记到本地报告工作区。GitHub Release 与互联网部署由仓库维护流程处理；正式报告持续遵循 AI trace、schema、确定性计算、证据校验和质量门禁。

## 适用场景

Asteria 适合结构化数据已经存在、但分析交付仍然分散的场景，例如：

- 月度、季度或专题经营复盘；
- 销售、运营、投放、库存、履约或门店数据诊断；
- 电商、采购销售或多工作表业务数据的结构化分析；
- 研究统计、方法比较和图表复核；
- 咨询交付中的多读者材料准备与后续版本维护。

系统会根据数据质量、口径、样本覆盖和业务背景呈现可支持的分析范围，并将待验证判断明确标记出来。

## 可复核性原则

公开版采用以下分工，明确生成、计算和发布职责：

| 事项 | 负责层 | 发布前校验 |
| --- | --- | --- |
| 字段含义、业务上下文、分析规划 | AI 辅助层与用户确认 | 生成结构化建议，数值由确定性执行器计算 |
| 数值计算与统计执行 | 确定性执行层 | 执行可复核计算 |
| 证据与报告绑定 | 证据校验、绑定和渲染层 | 绑定已校验的字段与业务上下文 |
| 正式发布 | FormalPDFReleaseGate | 审查 trace、schema、证据和质量条件 |

正式管理报告的完整规则见 [报告可信机制](report-integrity.zh-CN.md) 和 [AI 强制发布链](architecture_ai_mandatory_chain.md)。

| R 工作流 | 报告工作流；本机可解析 <code>Rscript</code> | 启动 R 数据准备、统计、可视化和工件关联 | 可能返回 CSV、XLSX、Markdown、HTML、PDF 或 R 工件索引 | 由本机 <code>Rscript</code> 路径、数据和报告请求共同启用。 |

## 隐私与公开边界

本项目的 GitHub 公开范围是源码、测试、启动脚本、示例和公开技术文档。下列内容保留在受保护的本地位置：

- 客户数据、个人信息、合同限制数据和真实业务报表；
- API Key、访问令牌、内部地址、运行日志和运行会话；
- 未授权的第三方材料；
- 私有作品集、历史交付物或能够反推客户身份的信息。

基础 CSV/Excel 分析可以在本机执行。若用户主动启用 AI 辅助报告或修订，相关字段上下文、报告上下文或上传的工作簿/PDF 可能会发送到用户配置的 OpenAI-compatible 提供方。处理敏感或受监管数据前，应先脱敏并遵循组织政策。

## 当前公开交付清单之外

- 公网多用户 SaaS、身份认证、租户隔离或合规托管需要独立部署项目；
- 外部模型、运行时、R 环境和扩展技能由本机配置和实际依赖决定；
- 客户成果、准确率、节省时间和规模数字待公开验证案例支持；
- Analysis Lab 输出进入正式管理报告前执行证据和质量门禁。
