# Asteria Analyst Windows 便携版离线指南

本文件会随 `AsteriaAnalyst-portable.zip` 一起提供，供 Windows 本机使用者阅读，无需预装 Python 或 Node.js 开发环境。它涵盖单用户、回环地址和本地文件处理流程。

## 1. 你需要什么

- Windows 电脑；
- 从项目 GitHub Release 下载的完整 `AsteriaAnalyst-portable.zip`；
- 一个可写、私有、空间充足的解压目录；
- 可选：用于 AI 辅助功能的、经过授权的本机环境变量配置；
- 第一份建议使用小型、无敏感信息的 CSV 或 Excel 样本。

基础数据导入、画像和受支持的本地分析可直接使用。AI 增强功能通过本机 API Key 配置启用。

## 2. 下载与完整性检查

1. 在对应 GitHub Release 页面下载 `AsteriaAnalyst-portable.zip`。
2. 若 Release 提供 SHA-256，下载后在 PowerShell 执行：

   ```powershell
   Get-FileHash .\AsteriaAnalyst-portable.zip -Algorithm SHA256
   ```

3. 比较输出的哈希与 Release 给出的完整 SHA-256。文件名和哈希都一致后再解压。
4. 将 ZIP 完整解压到例如 `D:\Apps\AsteriaAnalyst` 之类的可写私有目录。

不要在压缩包预览器中直接启动，也不要把包解压到 `Program Files`、公共共享目录、临时下载预览目录或受云同步共享的敏感工作目录。

## 3. 一键启动

在解压后的根目录双击：

```text
start-asteria.bat
```

启动器会：

1. 使用包内 Python 运行时启动后端；
2. 仅监听 `127.0.0.1`；
3. 从 `8787` 开始寻找一个可用本机端口；
4. 等待健康检查通过；
5. 自动打开 `http://127.0.0.1:<实际端口>/analysis`。

第一次启动请等待最多约一分钟。若 `8787` 被其他应用占用，启动器会尝试之后的端口并使用实际端口打开浏览器；不要手工假定地址始终是 8787。

后台服务由启动器独立管理。再次双击启动器时，它会优先复用受其管理的健康服务。需要升级、删除解压目录或排障时，先确认 Asteria 便携后端进程已经停止。

## 4. 完成第一次本地分析

1. 打开 `/analysis` 页面。
2. 上传 `examples/revenue-smoke.csv`（若包中提供）或一个小型、无敏感信息的 CSV/XLSX 文件。
3. 对于多工作表 Excel，先选择正确的工作表。
4. 阅读数据画像：行列数、字段类型、缺失、单位、时间范围和样本预览。
5. 用一句完整的话写清业务问题，例如“比较不同月份的收入变化并找出需要进一步核查的异常”。
6. 运行一个基础统计或进入 `/lab` 对比方法。
7. 先检查计算结果、表格和图表，再阅读或分享文字摘要。

上传完成后确认字段含义、行粒度、单位和业务解释。尤其在订单、客户、商品、门店和日期之间切换时，要先确认每一行代表什么。

## 5. AI 功能的配置方式

便携包的界面和公开 API 当前**不提供网页写入 API Key 或 Runtime 设置的功能**。便携版启动脚本也不自动读取你在源码版使用的 `.env` 文件。

若确实需要 AI 辅助：

1. 先确认数据可以发送给你选择的 OpenAI-compatible 提供方；敏感数据先脱敏或不要启用 AI。
2. 在 Windows 的“用户环境变量”或由受信任管理员管理的进程环境中设置 `OPENAI_API_KEY`；按需设置 `OPENAI_BASE_URL`、`OPENAI_MODEL` 和 `OPENAI_REASONING_EFFORT`。
3. 关闭并重新打开 `start-asteria.bat`，让新的进程读取环境变量。
4. 仅通过本机 `GET /api/runtime-settings` 的掩码摘要确认是否已加载密钥；不要试图在浏览器中查看或保存原始密钥。

不要把密钥写入 ZIP、便携包根目录文本、截图、GitHub Issue、报告附件或公开 Release Notes。

## 6. 本地数据保存在哪里

便携启动器默认把数据和私有设置放在：

```text
%APPDATA%\AsteriaAnalyst
```

这通常包含上传数据集、运行状态、私有设置、历史材料和公开工件子树。它不会因你删除解压后的程序目录而自动删除；反过来，覆盖升级程序目录也不保证数据可恢复。

建议：

- 在升级、清理或迁移电脑前，先关闭正在运行的便携版；
- 将整个 `%APPDATA%\AsteriaAnalyst` 备份到受保护的本机或加密位置；
- 不要把这个目录同步到公开网盘、提交 Git 或作为静态网站目录；
- 只在确认新版本可以读取所需数据后再删除旧备份。

## 7. 可选 Runtime 的边界

| 能力 | 便携版默认行为 | 需要什么 |
| --- | --- | --- |
| 基础本地分析 | 可用 | 无 API Key。 |
| AI 辅助 | 未配置时不可用 | 受信任的本机环境变量和数据授权。 |
| R 智能流 | 取决于包内或本机可用的 `Rscript` | Release 说明应明确是否包含 R 运行时。 |
| Codex Runtime | 默认关闭 | 本机管理员显式设置 Runtime 开关和受信任 CLI/工作区。 |
| 外部 Skill/Agent Team | 默认关闭 | 管理员显式允许本机安装，并审阅导入来源。 |
| 非沙箱执行 | 默认关闭 | 高风险本机管理员配置；不属于普通便携版使用流程。 |

不要为了让功能“看起来更多”而启用高权限 Runtime。当前便携版没有多用户授权和工作区隔离，所有这些能力都只适用于你自己受信任的机器和目录。

## 8. 常见问题

### 双击后浏览器没有打开

先等待一分钟，然后在浏览器访问：

```text
http://127.0.0.1:8787/analysis
```

若该地址无法访问，检查包内：

```text
backend\desktop-start.out.log
backend\desktop-start.err.log
```

日志中若显示端口调整，请使用日志或启动脚本显示的实际端口。公开求助时不要完整粘贴含路径、密钥或数据内容的日志。

### 提示启动超时或端口被占用

关闭或重新配置冲突应用后重试。启动器会在 8787 之后尝试有限范围内的本机端口；不要终止不明进程，也不要通过开放防火墙解决端口问题。

### 页面打开但无法分析

访问：

```text
http://127.0.0.1:<实际端口>/health
```

若没有返回 `{"status":"ok"}`，先检查 `desktop-start.err.log`。若健康检查正常，重新确认上传文件、工作表、字段类型、行粒度和方法要求。

### AI 功能不可用

基础分析不依赖 AI。若需要 AI，确认 Windows 用户环境中已设置正确的密钥/提供方变量，并在设置后重新启动便携包。不要把“配置了 API Key”理解为自动获得正式 PDF 发布资格。

### 正式报告无法发布

检查 AI trace、schema、确定性数值、证据校验和 `FormalPDFReleaseGate`。正式 `management_report.pdf` 需要全部检查项通过且最终质量分达到 90。

## 9. 升级、卸载与清理

### 升级

1. 备份 `%APPDATA%\AsteriaAnalyst`。
2. 下载新 Release 并核对 SHA-256。
3. 解压到新目录，不覆盖正在使用的旧目录。
4. 启动新版本并完成一次健康检查和无敏感样本冒烟。
5. 确认数据与关键功能正常后，再决定是否删除旧程序目录或旧数据备份。

### 卸载

先结束确认属于当前便携目录的 Asteria 后端进程，再删除解压目录。若要删除所有本地数据和设置，另行删除 `%APPDATA%\AsteriaAnalyst`；该操作不可恢复，先确认没有需要保留的数据或报告。

## 10. 使用边界回顾

- 便携版面向 Windows 本机、单用户和回环地址使用。
- 基础本地分析可以一键启动；AI 与高权限 Runtime 通过本机配置启用。
- 数据、密钥、报告、日志和用户数据目录都属于私有内容。
- 页面中的“发布”登记本地报告版本；GitHub Release 由仓库维护流程完成，正式 PDF 通过强制质量门禁后交付。
