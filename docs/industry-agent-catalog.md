# 行业 Agent 路线目录

> **定位**：本页定义 Asteria Analyst 的行业任务目录，用于规划业务对象、字段语义、分析方法和报告模板。每条路线都以“数据输入 -> 分析处理 -> 业务交付”为主线组织。页面中的实际路由状态由数据特征、任务配置和运行状态共同决定。

[功能逐项参考](function-reference.zh-CN.md) | [模块与功能手册](module-guide.zh-CN.md) | [详细用户指南](user-guide.zh-CN.md) | [Analysis Lab](page-results.zh-CN.md#lab---分析实验室)

## 1. 路线怎样进入分析任务

| 步骤 | 用户准备 | 系统组织 | 可审阅结果 |
| --- | --- | --- | --- |
| 选择场景 | 业务问题卡、数据集、时间范围、目标读者 | 用数据画像和字段语义识别候选路线 | 场景建议、字段角色、数据质量提示 |
| 定义对象 | 指标、维度、分组、时间和业务对象 | 绑定对象本体、口径和方法参数 | 可复用的字段绑定与方法配置 |
| 运行分析 | 数据范围、比较问题、方法与阈值 | 执行受控统计、图表和证据组织 | 结果表、图表、方法说明、证据资产 |
| 组织交付 | 读者、交付深度、历史材料和业务约束 | 生成相应的报告结构与行动清单 | 摘要、业务发现、待验证问题和下一步动作 |

正式 <code>management_report.pdf</code> 沿用项目的 AI 强制发布链：字段语义、业务路由、指标规划、确定性计算、证据校验、报告绑定和 PDF 门禁依次完成。详见[正式报告可信机制](report-integrity.zh-CN.md)。

## 2. 通用横向 Agent

横向 Agent 按经营问题复用，可跨行业接入。表中的“建议方法”表示适合在 Analysis Lab 中配置和审阅的方法族。

| Agent | 核心对象与建议输入 | 分析处理 | 业务交付 |
| --- | --- | --- | --- |
| 投放复盘 | 媒体投放明细、Campaign 日志、广告位、预算执行；曝光、点击、完成率、CPM、CPC | 预算与效果分布、窗口波动、偏差贡献、分组比较 | 投放效率分层、异常窗口、预算调整优先级 |
| 销售经营 | 订单明细、SKU/SPU、渠道销额、价格带；销售额、销量、客单价、份额、动销率 | 增长拆解、结构变化、渠道与品类切片、聚类或分群 | 增长来源、重点品类、渠道动作清单 |
| 价格促销 | 价格表、活动表、折扣规则、价格带历史；价格、折扣率、毛利、促销 uplift | 价格带迁移、活动组对照、差异检验、回归与结构分析 | 促销复盘、价格带建议、利润影响说明 |
| 消费者研究 | 问卷、NPS、满意度、访谈编码、受众细分；构念得分、偏好、选择率 | 人群分层、量表比较、卡方或秩和检验、PCA、聚类 | 受众画像、驱动因素、研究结论与追访问题 |
| 品牌舆情 | 帖子、评论、品牌提及、情绪标签、事件时间线；提及量、情绪占比、主题密度 | 主题与情绪归类、峰值追踪、事件关联、风险表达汇总 | 话题演化、风险清单、沟通和内容建议 |
| 渠道复盘 | 渠道销额、门店面板、区域表现、库存与动销；单店产出、份额、增长率 | 渠道/区域切片、结构分解、分组检验、异常门店定位 | 渠道效率地图、重点区域、门店或库存复核项 |
| 实验评估 | A/B 结果、实验分组、前后数据、增量测量；uplift、置信区间、效应量、功效 | 实验分组校验、差异检验、效果量、分层结果与复验设计 | 实验结论、放量条件、复验方案和指标看板 |
| 财务预算 | 预算执行、成本中心、利润表、费用分项、期间记录；收入、成本、利润、偏差额 | 预算偏差拆解、趋势、敏感点和成本结构分析 | 偏差来源、预算动作、利润敏感项 |
| 公益基金会 | 项目清单、收支、服务领域、地点、年度财务；项目集中度、领域占比、投入产出 | 项目结构、地域和领域分布、集中度、重点项目分层 | 资源投向地图、重点项目清单、项目治理问题 |

## 3. 行业纵向 Agent

纵向 Agent 将通用分析能力与行业对象、指标字典和交付模板组合。每条路线可从同一套数据接入、字段画像、方法卡和报告资产继续扩展。

| Agent | 核心数据与对象 | 重点分析任务 | 典型交付 |
| --- | --- | --- | --- |
| 互联网运营 | 用户行为、内容、拉新/活跃/留存、活动、漏斗、社区互动；用户、内容、活动、渠道 | 漏斗、留存 cohort、内容分发、活动效果、用户分层与窗口波动 | 运营日报/周报、活动复盘、内容表现、留存诊断、漏斗动作 |
| 电商零售 | 商品、订单、会员、活动、流量、转化、客服、退货；商品、店铺、会员、活动 | 商品结构、活动效率、转化瓶颈、复购与会员价值 | 商品经营盘点、活动复盘、转化链路、会员分层 |
| 快消品牌 | 投放、渠道分销、销量、价格、促销、舆情、消费者研究；品牌、品类、渠道 | 品牌心智、货盘、渠道推进、传播与销量联动 | 品牌经营摘要、渠道推进建议、传播复盘、价格促销专题 |
| 教育培训 | 课程、学员、转化、续费、出勤、评价、教学服务；课程、班级、学员、渠道 | 课程转化、完课与续费、学员分层、渠道效率 | 招生转化看板、续费诊断、课程运营行动清单 |
| 医疗健康 | 科室运营、患者流量、复诊、治疗项目、费用、药品；科室、项目、就诊路径 | 科室增长、复诊、费用结构、项目组合与服务容量 | 科室经营画像、服务路径复核、项目组合与风险提示 |
| 金融业务 | 贷款、信用卡、理财、风控、生命周期、渠道获客；客户、产品、渠道、风险组 | 获客质量、转化、流失或违约风险、资产和收益结构 | 客户生命周期、渠道质量、风险分层、经营组合复盘 |
| 制造供应链 | 产线、库存、出入库、采购、交付、良率、成本；工厂、产线、物料、订单 | 库存健康、交付波动、产能利用、良率和成本敏感点 | 供应链运行摘要、瓶颈清单、库存与交付行动项 |
| 本地生活/服务 | 门店、交易、评价、区域投放、到店转化、履约；门店、区域、服务单 | 门店差异、区域结构、履约质量、增长机会 | 门店经营地图、区域策略、履约改进清单 |
| 游戏与内容社区 | 内容分发、活跃、留存、付费、创作者、活动；玩家、内容、创作者、社群 | 内容爆发、社区活跃、留存、付费渗透、创作者结构 | 内容生态盘点、留存与付费专题、创作者运营动作 |
| B2B/SaaS | 线索、商机、销售漏斗、客户成功、续费、功能使用；账户、商机、用户、产品 | 线索质量、商机转化、客户留存、续费、产品使用深度 | 销售漏斗、账户健康度、续费预测输入、产品采用复盘 |

## 4. 每条路线的接入清单

| 层次 | 需要补齐的内容 | 形成的可复用资产 |
| --- | --- | --- |
| 任务建模 | 路线名称、业务问题模板、目标读者与交付类型 | 任务族与报告简报模板 |
| 对象与字段 | 行业对象、主键粒度、指标口径、时间字段、维度关系 | 字段语义词典、对象本体、字段绑定规则 |
| 方法与证据 | 推荐方法、前提条件、比较维度、阈值与审阅问题 | 方法卡、结果表、图表和证据清单 |
| 报告与行动 | 摘要结构、业务发现、待验证解释、行动项和复盘周期 | 行业报告模板、行动清单、复盘框架 |

## 5. 实施顺序

### 第一批：复用度最高

1. 投放复盘 Agent
2. 销售经营 Agent
3. 互联网运营 Agent
4. 公益基金会 Agent
5. 实验评估 Agent

这组路线共享投放、经营、用户行为、实验和结构拆解能力，适合作为字段语义和方法卡的首批验证场景。

### 第二批：经营分析扩展

1. 价格促销 Agent
2. 消费者研究 Agent
3. 品牌舆情 Agent
4. 渠道复盘 Agent
5. 财务预算 Agent

### 第三批：行业对象深化

1. 电商零售 Agent
2. 快消品牌 Agent
3. 教育培训 Agent
4. 医疗健康 Agent
5. 金融业务 Agent
6. 制造供应链 Agent
7. 本地生活/服务 Agent
8. 游戏与内容社区 Agent
9. B2B/SaaS Agent

## 6. 统一验证方式

每条新路线都使用同一套验证顺序：准备匿名样例数据，完成数据画像和字段映射，运行代表性方法，审阅结果资产，再用目标读者视角检查报告结构与行动项。正式管理报告进入完整 AI 强制发布链，并由质量门给出发布结果。
