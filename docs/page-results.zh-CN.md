# Asteria Analyst 页面结果说明

本文按当前前端实际路由说明用户在每个页面完成操作后能够看到、获得和继续使用的结果。它描述的是已实现的界面与接口行为，不把目录中的规划能力、可选扩展或后端未返回的产物写成默认可用功能。

## 先读这份说明

- “完成”表示当前页面的请求、任务或交互已经由系统返回成功状态；它不等同于分析结论已经通过业务审核，更不等同于正式管理报告已经获准发布。
- 下载入口、文件链接、图表、报告分节、运行时状态和扩展能力都由后端实际返回决定。页面不会凭空生成未返回的文件或结果。
- 正式【management_report.pdf】必须遵循受控链路：【raw data -> DataProfileService -> AIFieldSemanticMapper -> AIBusinessContextRouter -> AIMetricDerivationPlanner -> DeterministicMetricExecutor -> EvidenceValidator -> ReportBindingLayer -> FormalPDFReleaseGate -> management_report.pdf】。缺少 AI 追溯、Schema 校验、确定性数值执行、证据校验或发布门禁批准时，只能视为调试、预览或普通下载结果，不能视为正式管理报告。
- 所有需要读取数据、提交任务、获取报告或加载修订会话的页面都依赖可访问的后端 API。前端会显示加载、失败或重试状态；无法连接后端时不会伪造成功结果。
- GitHub 仓库或 GitHub Pages 上的静态文件本身不能替代后端服务。要实际使用页面，必须按启动文档运行后端并让前端的 API 地址指向它；仅打开静态网页时，数据、报告和修订结果不会可用。

## 页面总览

| 路由 | 页面定位 | 用户完成后的主要结果 |
| --- | --- | --- |
| / | 工作台入口 | 进入与分析页相同的智能报告工作区，并看到公开发布版的运行时配置边界说明。 |
| /analysis | 智能报告工作台 | 上传或选择数据并创建报告任务，查看进度、报告正文、关键指标和后端返回的下载产物。 |
| /lab | 分析实验室 | 配置数据、字段和方法后执行自动、单方法、批量或报告试验，查看统计、报告或通用运行结果。 |
| /lab/method-guide | 方法引导 | 完成四章方法配置学习路径，得到可带回分析实验室执行的操作要点，不产生分析文件。 |
| /revision | 报告修订库 | 浏览已有报告，打开报告详情，并创建或恢复可用的修订会话。 |
| /revision/workspace | 报告修订工作区 | 在指定报告和会话中查看修订过程、文件差异、批注、附件、运行状态及本地修订发布结果。 |

## / - 工作台入口

### 入口与用途

访问根路径后，页面提供进入分析工作流的主入口，并加载智能报告工作区。它与【/analysis】使用同一套报告创建与结果展示能力，同时增加公开发布版的运行时说明面板。

### 完成后看到的结果

- 当数据集、历史报告和业务背景资源加载成功后，可在工作台内选择现有资源或上传新资源。
- 创建报告任务后，页面显示任务阶段、进度百分比、阶段事件和最终任务状态；成功完成时展示报告结果。
- 页面还明确显示：公开发布版不在浏览器中保存或输入 API 密钥、模型提供方配置或执行路径。这是运行边界说明，不是可编辑的在线设置页。

### 主要产物

- 已上传或已选择的数据集、工作表、历史报告素材和业务背景素材。
- 报告任务返回的标题、摘要、关键指标、章节、表格、可下载文件及可选的 AI/Codex 辅助信息。
- 仅当后端返回主下载文件或其他下载文件时，界面才会显示对应入口。

### 下一步

- 需要直接使用完整报告工作流时，继续在当前页面操作，或进入【/analysis】。
- 需要比较方法、调试字段绑定、执行多种统计方法时，进入【/lab】。
- 已有可修订报告时，进入【/revision】从报告库中选择对象。

### 空态与失败边界

- 没有数据集、没有选中工作表或后端资源未加载成功时，不能开始有效报告生成。
- 后端连接失败、上传失败或任务失败时，页面显示对应状态或错误信息，不会生成虚假的报告结果。
- 运行时说明面板不提供浏览器端密钥填写、保存或转发能力；部署配置必须在服务端环境中完成。

## /analysis - 智能报告工作台

### 入口与用途

该页面用于从数据集、工作表、业务问题、目标读者、报告风格、历史报告素材和业务背景素材中组织一次智能报告任务。用户也可以在同一页发起快速统计分析。

### 完成后看到的结果

- 数据集选择成功后，可查看数据集概览，并根据实际数据选择一个或多个工作表。
- 上传成功后，数据集、历史报告或业务背景素材会成为可选择资源；上传状态和失败原因会在界面中反馈。
- 报告任务创建后，页面轮询任务状态，展示阶段进度、阶段事件和完成/失败状态。
- 报告完成后，结果区会展示报告标题、来源数据集、工作表、生成时间、关键指标、执行摘要、章节摘要、要点和可展示表格。
- 任务显示 `completed` 只表示该智能报告任务已经结束；当前页面不会把这一状态显示为正式 PDF 放行结论，不能据此跳过后续的证据与发布门禁核验。
- 后端提供 AI/Codex 辅助内容时，结果区还会展示执行摘要改写、战略建议、风险提示或待回答问题；未返回时不会显示为既有结论。
- 使用历史报告素材时，后端若返回模板信号、适配说明、报告族信息、计划/已渲染页数、图表数或表格数，页面会展示这些适配结果。
- 发起快速统计后，页面展示统计标题、文字说明和后端返回的部分指标，便于在正式报告前快速检查数据问题。

### 主要产物

- 报告任务记录及其状态、阶段事件和报告结构化结果。
- 后端返回的主下载文件与其他下载文件；每个文件只在实际返回路径和用途时显示。
- 报告章节中的摘要、要点和表格预览。表格预览是阅读界面，不能替代完整原始数据或原始工作簿。
- 可选的历史样式适配信息和 Markdown 预览。

### 下一步

- 审阅摘要、指标、表格和下载产物，确认其是否满足业务问题、目标读者和范围约束。
- 需要验证统计方法、字段绑定或不同方法输出时，带着相同数据进入【/lab】。
- 报告已经保存并需要继续修订、添加附件或查看差异时，进入【/revision】。
- 准备将结果作为正式管理报告发布前，必须确认正式 AI 链路和【FormalPDFReleaseGate】均已通过；普通预览、调试报告或下载文件不自动获得正式资格。

### 空态与失败边界

- 必须至少选择一个数据集和一个工作表；缺少任一项时，页面会阻止报告任务提交。
- 数据集详情仍在加载、资源初始化失败、上传失败或报告任务失败时，用户只能根据提示重试、调整输入或检查后端服务。
- 报告内容、表格、下载文件和 AI/Codex 说明均是可选返回项。没有对应返回值时，页面只显示当前可用结果，不补造内容。
- 快速统计是结果检查入口，不会绕过正式报告的 AI 追溯、确定性计算、证据校验和发布门禁要求。

## /lab - 分析实验室

### 入口与用途

分析实验室用于更细地控制数据、字段、方法、参数和运行队列。它适合在生成报告前进行探索、方法比较、字段映射检查、批量试验和运行结果审阅。

### 完成后看到的结果

- 选择或上传数据集后，页面加载数据摘要、字段信息及可用于方法配置的上下文。
- 自动运行、单方法运行、批量运行和报告试验会生成独立的运行记录。运行记录保留方法名称、运行状态、运行时间、执行器/任务信息、清单信息和错误信息等可用字段。
- 返回统计结果时，结果画布展示统计型结果；返回智能报告结构时，展示报告型结果；其他合法返回则以通用结果视图展示。页面不会把不同结果类型强行伪装成同一种报告。
- 当运行返回确定性管理报告交付包时，页面会显示运行状态、`quality_status`、通过/失败检查数、报告 ID、报告分节/方法/图表数量、证据覆盖引用、质量摘要，以及后端实际返回的 HTML、Markdown、修订种子和修订工作区入口。
- `quality_status=failed` 时，页面会明确把该结果标记为不能作为最终交付；若没有可用 HTML 预览，则应修正阻断项后重新运行，而不是把下载列表中的其他文件当作正式报告。
- Smart Merge 只用于增强或合并可用结果，不能替代确定性报告质量闸门，也不能把实验室结果提升为正式管理报告。
- 运行过的方法会进入方法运行列表，便于区分已执行、运行中、被阻止、需要运行时条件或出现错误的项目。
- 某些方法、报告分节或字段绑定可以在调整后重新运行。重新运行的目标是刷新该次试验结果，不是自动修改历史正式报告。
- 后端返回文件、JSON 结果或运行时产物时，页面提供相应查看或下载入口；没有返回时，不显示不存在的导出文件。

### 方法状态与可执行性

- 方法目录中的【live】表示当前具备直接运行条件。
- 【catalog】表示可在目录中查看或选择，但不代表已经具备完整执行条件。
- 【planned】表示规划中的能力，不能当作当前可执行结果。
- 页面会依据数据集、字段、方法参数、运行时条件和忙碌状态阻止无效提交，并给出可见的阻止原因或状态。

### 可选扩展结果

- 外部 Skill、Feature Trial 和 Report Agent Team 是已提供的可选本地扩展，不是预告功能。读取列表或目录时，页面可能显示空状态；安装、导入、挂载、取消挂载、删除、试验和团队启动则需要可信本机启用 `ASTERIA_ENABLE_LOCAL_SKILL_INSTALLER=1`。
- Skill 面板会显示已安装/已挂载数量、来源、插件信息、内嵌 Skill、命令和 MCP 连接器摘要。挂载的技能说明会以受限上下文写入本次 Lab 输出的 `external_skill_context.json`；页面不会把包中声明的命令或 MCP 配置自动当作已执行结果。
- 运行一次 Feature Trial 后，页面会得到 `trial_id`、数据集/插件/功能信息、基线数据画像、就绪度评分和原因、重点字段、推荐动作、建议的 Lab 运行参数以及可用下载链接。对应工件为 `trial.json`、`field_scores.csv` 和 `trial_report.md`，位于本次试验的存储目录；试验是数据适配性评估，不是任意第三方命令执行。
- Report Agent Team 面板支持导入用户自有的本机 Markdown 团队、挂载/取消挂载和删除。启动前必须选定数据集并至少挂载一个团队；任务创建后页面显示团队数量、已挂载数量、任务 ID、任务状态和工作区路径。后端将角色说明写入受控工作区后交给单个 Codex 任务，不承诺后端并行多 Agent 调度。
- Team Run 的入口需要本地扩展安装器开关，实际执行还需要可用 Codex CLI、受限工作区和 Runtime 条件。任务成功、失败、超时或取消时，页面应按后端状态显示摘要、差异、日志/工件链接或错误，而不能把排队状态误写成已完成。
- Lab 的 Report Agent Team 与修订工作区的 Report Agent Session 不同：前者为新任务准备团队角色和数据上下文，后者管理已有报告的消息、附件、批注、事件和差异。两者都不等同于正式 `management_report.pdf` 发布。
- 具体的启用步骤、请求字段、运行产物和故障处理见 [本地扩展指南](local-extensions.zh-CN.md)。

### 下一步

- 对照方法运行记录检查字段选择、目标变量、分组变量、参数和错误信息，再决定是否重新运行。
- 将经过审阅的结果带入【/analysis】组织受控报告任务，或在实验室中继续比较其他可执行方法。
- 要发布正式管理报告时，仍须通过完整 AI 强制链路和发布门禁；实验室结果、方法卡和试验文件不能替代正式发布批准。

### 空态与失败边界

- 未选择数据集、数据集详情尚未加载、未选择单方法、批量模式未选择方法，或当前运行仍在忙碌时，页面会阻止启动。
- 字段条件不满足、方法运行时不可用、方法处于非【live】状态或参数不完整时，运行会被阻止或标记失败。
- 大型数据集的部分预处理可能被延后，页面会以状态或原因提示用户继续处理，而不是声称已完成全部准备。
- 外部技能和代理团队的安装、挂载、试验或运行失败只影响对应扩展，不会让页面把失败结果包装为常规分析成功。

## /lab/method-guide - 方法引导

### 入口与用途

该页面提供独立的方法配置学习路径，也可作为分析实验室中的方法引导入口。页面按对象、字段、实例和解释输入等主题组织四章内容，帮助用户理解一次方法运行需要准备什么。

### 完成后看到的结果

- 用户可逐章阅读目标、操作步骤、示例和检查点，并看到当前主题及 1/4 到 4/4 的导航进度。
- 走完引导后，页面给出可带回实验室使用的配置思路，例如先确定分析对象、再选择字段、配置实例并解释输出。
- 该页面的“完成”只代表当前浏览器中的学习和配置准备进度；它不是持久化的系统分析记录，也不是数据分析、统计计算或报告生成完成。

### 主要产物

- 浏览器中的章节进度、检查点和方法配置参考。
- 可复用的操作思路，用于回到【/lab】选择真实数据和可执行方法。

### 下一步

- 进入【/lab】，选择数据集、字段与方法并发起实际运行。
- 在实验室获得结果后，再根据业务需要进入【/analysis】组织报告，或进入【/revision】修订已有报告。

### 空态与失败边界

- 方法引导不读取数据集、不执行统计方法、不上传文件，也不生成下载文件或正式报告。
- 页面中的示例和检查点不能替代对实际数据字段、方法前提和业务语境的验证。
- 如果客户端资源未能加载，页面不能作为离线分析或报告生成的替代通道。

## /revision - 报告修订库

### 入口与用途

该页面是已有报告的修订入口。用户先在报告库中按关键字、数据集、排序或分页结果定位报告，再查看报告详情并打开、创建或恢复对应的修订会话。

### 完成后看到的结果

- 报告库加载成功后，页面显示当前可访问的报告列表及其可用元数据。
- 选择报告后，页面加载该报告详情，并在可用时恢复已有修订会话或创建新的修订会话。
- 会话可用时，页面进入报告修订工作台，显示会话时间线、运行状态、报告预览、文件、差异、批注、附件和事件等后端实际返回的内容。
- 会话创建不可用时，系统可以退回只读报告工作区：用户仍可查看已有报告内容，但不能把只读预览误认为可执行的 Codex 修订会话。

### 主要产物

- 已选报告的详情和预览上下文。
- 已创建或已恢复的修订会话标识、会话事件、对话轮次、文件清单、差异、批注和附件。
- 后端返回的运行进程状态和 Codex 健康状态，用于判断修订任务是否能够继续。

### 下一步

- 从报告库打开具体报告后，进入【/revision/workspace?report_id=<报告ID>&session_id=<会话ID>】继续会话型修订。
- 在工作台中审阅差异、附件和批注，再决定是否提交本地修订发布。
- 需要重新做数据分析或重建报告内容时，回到【/analysis】或【/lab】，不要将修订库当作数值重算的替代入口。

### 空态与失败边界

- 报告库为空时，没有可打开、可修订或可发布的报告对象。
- 报告列表、详情或会话请求失败时，页面显示加载/错误状态；用户应重试或确认后端服务、报告 ID 和权限状态。
- 会话、附件、差异或事件等属于可选后端资源。局部加载失败时，其他已加载区域可能仍可查看，页面会继续尝试刷新或回退。
- 此页面中的“发布修订”仅指系统内的报告修订版本操作，不是推送 GitHub、发布网站或公开部署。

## /revision/workspace - 报告修订工作区

### 入口与用途

该页面用于处理指定报告和指定修订会话。通常由报告修订库进入，并通过查询参数传入【report_id】和【session_id】。它把报告预览、会话时间线、修订文件、差异、批注、附件和运行状态集中在同一工作区。

### 完成后看到的结果

- 有效报告和会话加载后，用户可以查看报告详情、会话消息、事件流、当前运行进程以及 Codex 健康状态。
- 提交修订请求后，页面显示会话中的消息、任务进度和后端返回的修订过程；运行中的任务可以按可用接口取消或恢复。
- 有文件差异时，页面展示差异文件；有批注和附件时，页面展示并允许在会话上下文中管理这些证据材料。
- 新上传的 CSV、TSV、Excel、Markdown、文本、PDF 或 JSON 附件会作为修订补充材料进入后端会话。附件本身不自动修改原报告中的数值结论。
- 发起本地修订发布后，用户获得的是后端记录的报告修订版本结果，而不是 GitHub Release、代码仓库推送或公共网页发布。

### 主要产物

- 报告 ID、修订会话 ID、消息和事件记录。
- 会话返回的文件、差异、批注、附件、预览内容和可用的运行状态信息。
- 本地报告修订版本及其发布状态，前提是后端发布请求成功。

### 下一步

- 在差异、批注和附件均已审阅后，再执行系统内的修订发布。
- 需要离开当前会话时，返回【/revision】选择其他报告或恢复其他会话。
- 修订后的内容若要形成正式【management_report.pdf】，仍需满足 AI 强制链路、确定性数值、证据校验和【FormalPDFReleaseGate】；修订会话不能绕过这些要求。

### 空态与失败边界

- 查询参数缺失、无效，或对应报告/会话不可访问时，不能据此凭空建立报告；应返回【/revision】从可用报告库中选择对象。
- Codex 会话不可用时，页面可以退回只读预览。只读状态不支持将自然语言请求当作已经执行的修订。
- 运行或续跑 Codex 修订链路需要服务端显式启用 `ASTERIA_ENABLE_CODEX_RUNTIME_API=1`；该开关默认关闭。已生成的同源 HTML/PDF 报告预览不依赖此开关。
- 事件流连接异常时，页面会通过状态提示、重连或轮询继续获取可用事件；连接恢复前不能假定任务已完成。
- 上传附件受支持格式限制，且上传、删除、批注、取消、恢复和发布均需要后端成功响应。失败时，原报告和已有产物不会被页面静默替换。
- 图表和数值应由后端确定性逻辑生成或核验；修订界面不应静默覆盖已有报告数值。
